네이버 Push 2단계 - 우선순위 선택

어떤 기능부터 구현할까요? 현재 운영 상황에 맞게 선택해주세요.

A

배송 Push (송장 전송)

주문 발송 처리 시 네이버에 택배사 + 송장번호 자동 전송

장점

  • 셀러센터 수동 발송처리 제거
  • 주문 상태 자동 업데이트 (SHIPPED)
  • 고객에게 자동 배송 알림

참고

  • 택배사 코드 매핑 필요
  • ERP에 송장번호 입력 UI 필요
B

재고 Push (재고 동기화)

ERP 재고 변경 시 네이버 스마트스토어 재고 수량 자동 반영

장점

  • 품절/재입고 자동 반영
  • 과판매 방지
  • 공구 이벤트 재고 관리 편리

참고

  • SKU ↔ 네이버 상품 매핑 필요
  • 재고 계산 로직(HOK InventorySnapshot) 연동
C

둘 다 동시 진행

배송 Push + 재고 Push를 함께 구현

장점

  • 한번에 네이버 Push 완성
  • 운영 자동화 극대화

참고

  • 구현 범위가 넓어짐
  • 테스트/검증 시간 증가

현재 상태 참고

  • Pull 완료: 네이버 주문 549건, 상품 7종, 고객 연락처 동기화
  • Push 없음: 현재 셀러센터에서 수동 처리 중
  • 공구 시작: 4/6 예정 - 재고 관리 자동화 시 효과 큼
