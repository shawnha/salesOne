Key Design Decisions

  • company_id on every table — enables per-company filtering and group-wide rollups
  • Order.type = INTER_COMPANY — inter-company transfers are modeled as orders with a linked transfer record
  • Production orders are HOK-specific — but stored in the shared schema for group visibility
  • Customer.type — distinguishes D2C consumers (HOI) from drugstores (HOK/HOR)
